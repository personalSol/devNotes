- in aws you have three level for locations
- regions => availability zones => data centers
- each region has a minimum of 3 zones and max of 6 with each zone having around 1 to two data centers
- we can select services on basis of global and regional level
- global services are services which are available everywhere and has safe features and stuff for everywhere
- regional services are different for each region so if we dont find one service in our region then we have to select other region in which that service is available and also bear latency for that service
- it is also that selecting region also changes interface as well as things around service
- it is suggested to never use root user account for anything except for billing and account setting
